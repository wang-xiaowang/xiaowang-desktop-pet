# 桌宠-小旺动作素材说明

这份文档用于指导绘制小旺的动作素材。每一张图都要保持同一画布尺寸、透明背景、主体居中、脚底或身体底部基准线一致；同一组动作尽量基于同一张参考图只改局部，不要每帧重新设计姿势，否则切帧时会跳动。

当前代码会优先读取 `assets/body/` 和 `assets/transitions/` 下的 PNG 文件。文件名必须和下面的代码名称一致，例如 `idle_01.png`、`t_idle_read_01.png`。

## 身体循环帧

`idle_01`：端坐基础姿势，小旺是一只蓝白小猫，正面或略微 3/4 朝向屏幕，身体自然坐稳，尾巴轻轻贴在身体一侧或绕在前方，双爪放在身前，耳朵放松竖起，眼睛自然睁开，整体状态是安静陪伴、没有强烈动作；这是所有转场的起点，身体重心必须稳定，头部、躯干、脚底位置作为后续所有帧的对齐基准；

`idle_02`：端坐呼吸帧，整体姿势仍然和 `idle_01` 几乎一致，只让胸口和肩背有非常轻微的上浮或下沉，头部可以小幅度下压 1 到 2 像素，耳朵和尾巴保持几乎不动，表情仍然平静；这一帧要能和 `idle_01` 来回循环，形成细微呼吸感，不能像换了一个姿势；

`drag`：被拎起来的姿势，小旺身体被轻轻提起，四肢自然下垂或略微蜷起，耳朵可以微微向外，表情是有点无奈但不夸张；身体底部可以比端坐时高一些，但主体仍要在画布中央，不能超出画布；这个姿势用于鼠标拖动时瞬间切换，所以要明显区别于端坐，但不要有剧烈变形；

`walk_01`：走路第一帧，小旺从端坐变成站立后向右迈步，靠近屏幕前侧的一只前爪稍微抬起，另一只前爪支撑身体，后腿一前一后形成自然步态，尾巴轻轻向后平衡，头部保持稳定；这一帧要表现“刚迈出一步”，身体重心略微偏向支撑脚，但整体高度和 `walk_02` 接近；

`walk_02`：走路第二帧，小旺完成另一半步态，上一帧抬起的爪子落下，另一侧爪子准备抬起，身体重心轻微切换到另一侧，尾巴方向与 `walk_01` 有一点反向摆动；`walk_01` 和 `walk_02` 来回循环时应该像小旺慢慢散步，而不是跳跃或滑行；

`read_01`：看书循环第一帧，小旺从端坐进入看书状态后已经坐稳，双爪抱着一本小书，书本打开在胸前偏下的位置，头部微微低下看书，耳朵放松，尾巴保持安静；身体底部仍和端坐基准接近，只是上半身略向书本靠近，整体感觉是专注但舒适；

`read_02`：看书循环第二帧，小旺仍抱着同一本书，只做很轻微的翻页或眨眼动作；可以让一只爪子稍微抬起压住书页，书页边缘有一点变化，头部位置只允许轻微移动；这一帧要能和 `read_01` 平滑循环，像慢慢看书，不要突然抬头或大幅换书的位置；

`type_01`：敲键盘循环第一帧，小旺坐在原地，面前有一块小键盘或简化的键盘区域，身体稍微前倾，两只前爪放在键盘上方准备敲击，眼睛看向键盘或屏幕方向；头和身体重心要和活动状态一致，不能突然变高或变矮；

`type_02`：敲键盘循环第二帧，一只前爪下压按键，另一只前爪略微抬起，肩膀和头部随动作有很小幅度的前后变化，键盘位置保持固定；这一帧要和 `type_01` 连接成“开始敲”的动作，不能出现键盘漂移；

`type_03`：敲键盘循环第三帧，两只爪子的高低与 `type_02` 交换，另一只前爪下压按键，身体仍然轻微前倾，表情专注或认真；`type_01`、`type_02`、`type_03` 循环时要像左右爪交替敲键盘，节奏轻快但不夸张；

`ball_01`：玩小球循环第一帧，小旺坐着看向面前的小球，身体略微前倾，一只前爪伸出但还没有碰到球，另一只爪子支撑身体，尾巴轻轻翘起，眼神有一点好奇；小球主体可以用单独的 `ball_prop` 由代码控制，因此这一帧的身体动作重点是“准备拨球”；

`ball_02`：玩小球循环第二帧，小旺伸出的前爪向前轻轻拍到球的位置，肩膀略微前移，头部跟着爪子低一点，表情变得更专注；动作幅度要比 `ball_01` 明显，但身体底部仍保持稳定，不要像扑出去；

`ball_03`：玩小球循环第三帧，小旺拍完球后爪子略微收回，头部和眼睛追随球离开的方向，尾巴轻轻摆动；这一帧要能自然回到 `ball_01`，像“看球回来，再准备下一拍”，不能突然回正得太僵硬；

`ball_prop`：单独的小球道具，圆形或略带弹性的可爱小球，颜色与小旺蓝白配色有区分但不刺眼，透明背景，球体居中；这张图只画球，不画小旺，后续由代码控制弹跳或位移，所以边缘要干净，大小适合放在小旺爪子前方；

`sleep_01`：睡觉循环第一帧，小旺从端坐变成侧躺或趴睡姿势，身体蜷成舒服的一团，眼睛闭上，耳朵放松，尾巴靠近身体，整体重心比端坐低；姿势要安稳，像已经睡着，不要像摔倒；

`sleep_02`：睡觉循环第二帧，保持 `sleep_01` 的睡姿，只让胸口或背部有轻微呼吸起伏，尾巴尖可以小幅度动一下，眼睛仍闭着；这两帧循环时要非常慢、非常平静，不能出现明显抖动；

## 端坐到活动的转场帧

`t_idle_read_01`：端坐到看书的第一段过渡，小旺仍坐在 `idle_01` 的位置上，但一只前爪开始向身侧或身前取书，头部轻微低下，视线从正前方转向书本将要出现的位置；身体重心基本不变，动作感觉像“慢慢想起要看书”；

`t_idle_read_02`：端坐到看书的第二段过渡，小旺已经把书拿到胸前，书本半打开，双爪开始托住书，头部继续自然低下；这一帧要能接上 `read_01`，所以书本位置、爪子位置和 `read_01` 非常接近，只差一点完全坐稳；

`t_idle_type_01`：端坐到敲键盘的第一段过渡，小旺从端坐状态微微前倾，视线向前下方移动，前爪开始离开身体，像看到键盘准备靠近；身体底部仍不动，只有上半身和爪子开始变化；

`t_idle_type_02`：端坐到敲键盘的第二段过渡，小旺已经靠近键盘，两只前爪悬在键盘上方，头部和肩膀形成轻微前倾姿势；这一帧要自然接到 `type_01`，键盘位置需要和 `type_01` 完全一致；

`t_idle_ball_01`：端坐到玩小球的第一段过渡，小旺从端坐转为注意到前方小球，头部稍微偏向球的方向，一只前爪抬起一点，眼神从平静变成好奇；身体重心不要前扑，只表现“发现小球”；

`t_idle_ball_02`：端坐到玩小球的第二段过渡，小旺前爪继续伸向小球，身体轻微前倾，尾巴稍微翘起，眼睛锁定小球；这一帧要接近 `ball_01` 的准备拨球姿势，让进入玩球状态时不突兀；

`t_idle_sleep_01`：端坐到睡觉的第一段过渡，小旺从端坐开始犯困，眼睛半闭，头部轻轻低下，身体变软，耳朵略微放松；动作要慢，像困意上来，而不是突然倒下；

`t_idle_sleep_02`：端坐到睡觉的第二段过渡，小旺身体进一步放低，前爪收拢，尾巴靠近身体，已经接近趴睡或侧躺姿势；这一帧要能顺滑接到 `sleep_01`，身体落点和最终睡姿基本一致；

`t_idle_walk_01`：端坐到走动的第一段过渡，小旺从端坐准备起身，前爪撑地，身体略微抬高，后腿开始从坐姿调整为站姿，头部仍保持平稳；动作感觉是“慢慢站起来”，不能像瞬间弹起；

`t_idle_walk_02`：端坐到走动的第二段过渡，小旺已经基本站稳，身体朝向走路方向，第一只爪子准备迈出，尾巴轻轻向后保持平衡；这一帧要能自然接到 `walk_01`，高度、朝向和脚的位置要连续；

## 可选表情素材

当前版本代码默认关闭表情框，所以这一组可以先不画。等身体动作稳定后，如果要恢复表情，可以再单独设计更好看的气泡和符号。

`bubble`：空白云朵气泡，透明背景，边缘柔和，大小适合出现在小旺头顶，不遮挡主体；内部留出足够空间给代码贴 emoji 或小图标，气泡整体不要太亮或太厚重，避免抢走小旺本体的注意力；

`emote_heart`：爱心表情图标，表达小旺开心或亲近，形状圆润轻巧，颜色温暖但不要过饱和；如果以后启用，应能放进 `bubble` 中央，不要碰到气泡边缘；

`emote_note`：开心音符表情图标，表达小旺轻松愉快，音符可以有一点跳动感，但图形保持简洁；适合单击互动后的轻快反馈；

`emote_question`：疑惑表情图标，表达小旺歪头好奇，问号线条柔和，不要显得生气或惊恐；适合随机互动反馈；

`emote_anger`：轻微炸毛或不满表情图标，表达被打扰的小情绪，图形可以是小怒符但不要太凶；整体仍然可爱，不能破坏陪伴感；

`emote_sweat`：无奈汗滴表情图标，表达小旺有点尴尬或被拖起来后的无奈，颜色清爽，形状简单；不要画得像哭泣；

`emote_zzz`：睡觉专用表情，几个从小到大的 `Z` 或柔和睡眠符号，适合小旺睡觉时出现；线条要轻，不要吵醒感太强；
